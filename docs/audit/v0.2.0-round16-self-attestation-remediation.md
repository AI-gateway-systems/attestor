# v0.2.0 Round 16 Self-Attestation Remediation

Status: fixed on branch; not closed until PR merge, green checks, and
`origin/master` verification.

## Scope

Audit source: Round 16 self-attestation and one-engine guard verification.

Affected trust surface:

- shadow data quality producer trust
- signed assurance packet production signing status

Protected principles:

- proof integrity
- fail-closed boundary
- no overclaim
- auditability

## Findings

### B-054 - Producer-trust check fails open by default and warns only

Status: validated and fixed repo-side.

Repository evidence before fix:

- `shadow-data-quality-gate.ts` treated an empty `trustedProducers` allow-list
  as trusted.
- A configured but untrusted producer produced only a warning.

Risk:

A canonical shadow event with any producer string could become evidence material
without an explicit producer trust scope. This kept the producer boundary
partly self-attested.

### B-055 - Signature `productionReady` self-attestation

Status: validated and fixed repo-side.

Repository evidence before fix:

- `signed-assurance-packet.ts` normalized `signature.productionReady` directly
  from caller input.
- The production signing boundary allow-list constrained the effect, but the
  flag itself was still caller-declared.

Risk:

The signature status surface mixed a trusted boundary with a caller-provided
production flag. That made reviewer evidence harder to distinguish from
self-attestation.

## External Anchors

- NIST SP 800-115: technical security assessment findings should produce
  repeatable mitigation and verification evidence.
- OWASP ASVS: security controls should be verified against explicit trust
  boundaries and should not rely on unauthenticated caller-controlled claims.

Why applicable:

Both findings concern trust-boundary evidence. Producer trust and production
signing readiness are inputs to assurance material, so their status must be
derived from configured trust scope or known boundaries rather than unscoped
caller claims.

Why not overclaimed:

This remediation does not implement authenticated producer identity at the API
edge, does not complete customer-gate/shadow-chain binding, does not certify
external KMS/HSM integration, and does not prove production readiness. It only
removes two repo-side self-attestation paths.

## Smallest Safe Fix

- Treat missing `trustedProducers` scope as a failed producer-trust check with
  `producer-trust-unscoped`.
- Treat configured but unknown producers as failed checks instead of warnings.
- Derive signature `productionReady` from
  `SIGNED_ASSURANCE_PACKET_PRODUCTION_SIGNING_BOUNDARIES`, not caller input.
- Add regression tests for unscoped producer trust and boundary-derived
  production signing readiness.
